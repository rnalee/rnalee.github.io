hadoop安装

发表于 2019-04-05 | 分类于 hadoop,java
| 字数统计: 1.8k 字 | 阅读时长 ≈ 8 分钟

摘要：介绍hadoop单机模式、伪分布模式、完全分布模式三种模式的安装..

1.1 单机模式

1.1.1 安装前准备

（1）安装Ubuntu 16.04

​ 下载Ubuntu 16.04

（a）Desktop –> 桌面版，默认带了界面

​ ubuntu-16.04.5-desktop-amd64.iso

（b）Server –> 服务器版，默认没有带界面

​ ubuntu-16.04.5-server-amd64.iso

​ 打开VMware，点击文件（F）,新建虚拟机

​ 进入创建界面，选择经典（推荐）直接进入下一步

​ 填写登录用户及密码等，选择下一步

​ 自定义硬件配置后，选择完成

​ 进入虚拟机等待安装，最后直接输入用户密码即可登录

（2）关闭防火墙

查看防火墙状态，若为关闭状态，则无须关闭，若为开启状态，则须开启

sudo ufw status

Status: inactive

​

关闭防火墙

sudo ufw disable

防火墙在系统启动时自动禁用

（3）安装JDK

解压到根目录：

tar zxvf jdk-8u144-linux-x64.tar.gz -C ~

建一个软链接（方便使用,相当于Windows的快捷方式）

ln -s jdk1.8.0_144 jdk

配置环境变量：

vi ~/.bashrc

（等号两侧不要加入空格）

使得变量生效：（一定要生效，jdk才有用）

source ~/.bashrc

查看jdk是否配置成功：

java -version

（4）确认openssh-client、openssh-server是否安装

dpkg -l | grep openssh

​

如果没有安装，则安装：

sudo apt-get install openssh-client

sudo apt-get install openssh-server

（5）安装包解压

tar -zxvf hadoop-2.7.3.tar.gz -C ~

创建超链接：(便于使用)

ln -s hadoop-2.7.3 hadoop

配置环境变量：

vi ~/.bashrc

使环境变量生效

source ~/.bashrc

查看hadoop是否安装成功

hadoop version

1.1.2安装

（1）修改配置

vi hadoop-env.sh

export JAVA_HOME=/home/hadoop/jdk

（2）测试MapReduce程序

mkdir ~/input

随意增加测试数据：vim data.txt（如a，b，c，12，3，4，5，9，2）

运行：

hadoop jar hadoop-mapreduce-examples-2.7.3.jar wordcount ~/input/data.txt ~/output

1.2 伪分布模式

1.2.1安装前准备

（1）参考上面1.1.1

同1.1.1

（2）配置主机名

sudo vi /etc/hosts （前面加sudo，需要root权限）

191.168.1.51 node1.hadoop node1

（3）免密码登录

通过ssh-keyen生成一个RSA的密钥对

ssh-keygen -t rsa -P '' (一直按回车，用输入用户密码则输入即可)

公钥追加到~/.ssh/authorized_keys文件中

ssh-copy-id -i ~/.ssh/id_rsa.pub 主机名(如上面都nod1)

测试免密码登录：

ssh node1

1.2.2安装

（1）修改配置文件

hadoop-env.sh

export JAVA_HOME=/home/hadoop/jdk

hdfs-site.xml

dfs.replication 1

core-site.xml

fs.defaultFS hdfs://node1:9000 hadoop.tmp.dir /home/hadoop/hadoop/tmp

mapred-site.xml

该文件默认没有（cp mapred-site.xml.template mapred-site.xml）

mapreduce.framework.name yarn

yarn-site.xml

yarn.resourcemanager.hostname node1 yarn.nodemanager.aux-services mapreduce_shuffle

（2）格式化

hdfs namenode -format

（3）启动停止Hadoop的环境

start-all.sh

stop-all.sh

（4）测试

查看进程：

jps

有NameNode、 Secondary NameNode、DataNode、ResourceManager、NodeManager进程

通过本地浏览器：

HDFS:

http://主节点ip:50070

http://主节点ip:50090

Yarn: http://主节点ip:8088

通过MapReduce:

但预先要往hdfs传一个文件，如何操作? 1. Linux目录下，随意一个目录，创建一个文本文件data.txt，里面写些内容。 2. hdfs dfs -mkdir /input 3. hdfs dfs -put data.txt /input —上传 4. hdfs dfs -ls /input –查看

但预先要往hdfs传一个文件，如何操作?

  1. Linux目录下，随意一个目录，创建一个文本文件data.txt，里面写些内容。
  2. hdfs dfs -mkdir /input
  3. hdfs dfs -put data.txt /input —上传
  4. hdfs dfs -ls /input –查看

1.3 完全分布模式

安装规划

确定几个节点，Namenode、Datanode安装在哪几台机器上。

这里用两个Datanode，一个Namenode

每台机器参考上面“安装前准备操作”,同时增加下面几步：

2.1.1安装前准备

（1）参考上面1.1.1

同1.1.1

（2）配置主机名

sudo vi /etc/hosts

191.168.1.51 node1.hadoop node1 191.168.1.52 node2.hadoop node2 191.168.1.53 node3.hadoop node3

（3）安装ntp服务

sudo dpkg -l |grep ntp

若NTP尚未安装就运行安装命令：

sudo apt-get install ntp

注意：ntp服务需要注意能连接网络，如果时间不一样，执行MapReduce程序的时候可能存在问题

（4）配置免密码登录：两两之间的免密码登录

​ 每台机器上运行：

ssh-keygen -t rsa -P ''

ssh-copy-id -i ~/.ssh/id_rsa.pub node1

ssh-copy-id -i ~/.ssh/id_rsa.pub node2

ssh-copy-id -i ~/.ssh/id_rsa.pub node3

1.2.2安装

（1）修改配置文件

hadoop-env.sh

export JAVA_HOME=/home/hadoop/jdk

hdfs-site.xml

dfs.replication 2

core-site.xml

fs.defaultFS hdfs://node1:9000 hadoop.tmp.dir /home/hadoop/hadoop/tmp

mapred-site.xml (默认没有 cp mapred-site.xml.template mapred-site.xml)

mapreduce.framework.name yarn

yarn-site.xml

yarn.resourcemanager.hostname node1 yarn.nodemanager.aux-services mapreduce_shuffle

slaves

（2）格式化NameNode

hdfs namenode -format

（3）把主节点上配置好的hadoop复制到从节点上

scp -r hadoop-2.7.3/ lee@node2:/home/lee

scp -r hadoop-2.7.3/ lee@node3:/home/lee

（4）启动

start-all.sh

（5）测试

查看进程：

jps

通过Web界面：

HDFS: http://主节点ip:50070 http://主节点ip:50090

Yarn: http://主节点ip:8088

通过MapReduce:

hadoop jar hadoop-mapreduce-examples-2.7.3.jar wordcount /input/data.txt /output/
